# Citation Ledger Template

引用台账是报告可溯源的核心保证。每条引用必须有唯一编号，正文中的引用标注与台账中的记录严格一一对应。

## 编号体系

### 编号规则

```
C-{章节序号}-{流水号}
```

- 章节序号：用数字，如 C-3 表示第3章
- 流水号：三位数字，从001开始
- 同一来源多处引用时，使用字母后缀：C-3-012a, C-3-012b

### 特殊编号

| 前缀 | 用途 |
|---|---|
| C-0-xxx | 摘要/前言中的引用 |
| C-A-xxx | 附录中的引用 |
| C-E-xxx | 执行摘要（Executive Summary）中的引用 |
| C-T-xxx | 表格/图表专属引用 |

### 主表字段说明

| 字段 | 说明 |
|---|---|
| 引用编号 | 唯一编号，与正文标注一一对应 |
| 章节/位置 | 精确到段落，便于定位 |
| 事实陈述摘要 | 报告中引用的核心事实，20-50字 |
| 来源全称 | 完整来源名称，含文件名、文号等 |
| 来源类型 | 从类型码表中选择 |
| 发布日期 | 来源的发布日期或数据所属年份 |
| 可访问路径 | URL、内部文件路径、档案编号等 |
| 验证状态 | 从验证状态码表中选择 |
| 备注 | 时效提示、口径说明、验证备注等 |

### 来源类型码表

| 代码 | 类型 | 示例 |
|---|---|---|
| POLICY | 政策文件 | 国务院、发改委、能源局等发布的法规和规范性文件 |
| STATS | 统计数据 | 统计局、能源局、IEA等发布的统计数据 |
| CORP | 企业公开数据 | 年报、ESG报告、公告、官网信息 |
| REPORT | 行业/研究报告 | 国际组织、研究机构、咨询公司报告 |
| USER | 用户提供 | 内部数据、项目清单、规划稿等 |
| ACADEMIC | 学术文献 | 期刊论文、学位论文 |
| MEDIA | 权威媒体报道 | 新华社、人民日报等官方媒体报道 |
| OTHER | 其他 | 会议纪要、专家访谈等 |

### 验证状态码表

| 代码 | 含义 | 使用场景 |
|---|---|---|
| VERIFIED | 已验证通过 | 完成四层验证（存在性、时效性、准确性、语境）全部通过 |
| USER_PENDING | 待用户确认 | 用户提供的数据，需用户确认口径后采用 |
| PUBLIC_ACCESSIBLE | 公开可查 | 公开来源可定位到原文，但未逐字比对 |
| UNVERIFIABLE | 无法独立验证 | 非公开报告、内部讲话等无法通过公开渠道核实 |
| EXPIRED | 已过期 | 时效性验证不通过，标注"待更新" |
| FLAGGED | 引用存疑 | 准确性或语境验证不通过，标注"建议核实" |

| 优先级 | 定义 |

| P2 | 影响论据的充实程度，但不影响核心结论 |

## 台账交付要求

1. 台账与标注后的报告正文同时交付。
2. 台账独立成文件，或作为报告附录A。
3. 待补充引用清单（附录B）独立列出，按优先级排列。
4. 台账中每一条记录均可从正文中的 `[C-X-XXX]` 标注快速定位。
5. 用户提供的内部数据在台账中统一标注为"待用户确认口径"。
6. 交付前检查：台账记录数 = 正文中标注的总引用数（含多个子编号）。
