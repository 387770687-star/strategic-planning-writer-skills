# Source Verification Rules

确保每条引用真实存在、时效有效、语境一致。验证不通过的一律降级为 `[待补充引用]`。

## 来源分级：官方来源 vs 非官方来源

在验证前，先对来源进行分级。非官方来源不得作为最终引用依据。

### 可引用来源（白名单）

| 等级 | 来源类型 | 示例 |
|---|---|---|
| **S级：一手权威来源** | 政府公文、法律文件、央企年报/公告、国家统计局数据 | 国务院文件、国家管网年报、国家统计局年鉴 |
| **A级：官方机构和国际组织** | 部委网站、省级政府网站、央企官网、IEA/EIA/World Bank | 国家能源局数据、IEA World Energy Outlook |
| **B级：权威行业来源** | 行业协会、国家级研究机构、央企研究院 | 中石油经研院报告、中国石油和化学工业联合会 |
| **C级：可引用但需标注的** | 券商研报、咨询机构报告、行业会议公开报告 | 标注"第三方研究，不代表本报告立场" |

### 禁止引用来源（黑名单）

以下来源**绝对不得**作为数据或判断的引用依据：

- **商业新闻门户**：腾讯新闻、新浪新闻、网易新闻、搜狐新闻、凤凰网、澎湃新闻（非政策解读类）
- **自媒体/内容平台**：微信公众号、今日头条、百家号、知乎、雪球、微博、小红书、抖音
- **聚合类平台**：百度百科（作为唯一来源时）、搜狗百科、360百科
- **非官方数据聚合网站**：各类未注明原始出处的数据汇总页面

**处理方式**：此类来源中的数据如确需引用，必须追溯到原始发布机构的官方文件。新闻报道可作为"线索"指向原始出处，但不得作为最终引用来源。追溯失败的，标记为 `[待补充引用]`。

---

## 验证层次

采用四级验证框架，逐层递进：

```
存在性验证 → 时效性验证 → 准确性验证 → 语境验证
   ↓              ↓              ↓            ↓
 文件存在吗？   数据过期了吗？  数字对吗？    被曲解了吗？
```

---

## 一、存在性验证

### 验证问题

| # | 问题 | 不通过时处理 |
|---|---|---|
| 1 | 引用的文件/报告/讲话是否真实存在？ | 标记为 `[待补充引用]` |
| 2 | 发文机关/发布机构名称是否准确？ | 修正为官方全称 |
| 3 | 文号是否与公开信息一致？ | 标注"文号待核验" |
| 4 | 数据发布机构是否真的发布了该数据？ | 标记为 `[待补充引用]` |

### 存在性判断标准

**确认为存在**（可直接引用）：
- 能在政府网站、官方公报、权威数据库中查到的文件
- 企业官网/年报/正式公告中的信息
- 公认国际组织（IEA、EIA、World Bank等）的公开报告
- 用户提供的内部文件（标注用户来源）

**无法确认存在**（必须降级）：
- 网络上搜索不到文件原文或官方报道
- 文号不符合该机关发文编号规则
- 声称的讲话无法在公开渠道找到
- "据报道""有消息称""据内部人士"等模糊来源
- 来源为黑名单中的非官方渠道（腾讯新闻、微信公众号等）——此类即使内容存在，也不得直接引用

**灰色地带**（允许引用但需标注）：
- 行业会议/论坛上的报告（标注会议名称、时间、报告人）
- 咨询机构内部报告（标注"非公开报告，无法独立核实"）
- 征求意见稿/草案（标注"非正式版本"）

## 二、时效性验证

### 时效标准

| 引用类型 | 有效期限 | 超过期限的处理 |
|---|---|---|
| 国家级政策文件 | 5年（规划类至规划期末） | 标注"注意政策时效性，建议核实最新政策" |
| 部委规章 | 5年或至修订日 | 标注"注意时效，建议核实是否已修订" |
| 经济/市场统计数据 | 2年 | 标注"数据需更新至最近年份" |
| 技术参数 | 3年 | 标注"技术标准可能已更新" |
| 企业年度数据 | 1个财年 | 标注"需更新至最近财年" |
| 国际组织预测报告 | 报告发布后2年 | 标注"预测基准年已过，需更新预测" |
| 已废止/被替代的政策 | — | 禁止引用，替换为现行政策 |

### 时效检查清单

- [ ] 政策文件：检查是否有更新版本、替代文件或废止公告
- [ ] 规划文件：确认是否在规划有效期内
- [ ] 统计数据：确认是否为最新可获取年份
- [ ] 企业数据：确认财年是否已过，年报是否已发布
- [ ] 预测数据：确认基准年是否已过，预测是否需要刷新

---

## 三、准确性验证

### 数值类

| 检查项 | 方法 | 容差 |
|---|---|---|
| 绝对数值 | 与原文交叉比对 | 完全一致 |
| 增长率 | 与原文交叉比对，基期一致 | ±0.1个百分点 |
| 占比 | 与原文一致，分母统一 | 完全一致 |
| 单位 | 确认与原文单位一致，转换正确 | 不得有误 |
| 汇率换算 | 标注汇率基准和日期 | — |
| 四舍五入 | 标注"约"，以免过度精确 | — |

### 文本引用类

- 政策文件名称：与官方全称一字不差
- 引用原话：使用引号标识，与原文一字不差
- 概括性引用：不得改变原文核心意思
- 发文机关名称：使用规范的机关全称

### 禁止的准确性错误

- 将"征求意见稿"当作正式文件引用
- 将地方政策当作国家政策引用
- 将企业目标当作已完成事实引用
- 将预测结果当作确定数据引用
- 将"管道设计能力"混同"实际输量"
- 将"规划投资"混同"已完成投资"
- 将非官方新闻网站的报道数据当作权威来源引用
- 将自媒体/公众号文章中的行业判断作为正式引用依据

## 四、语境验证

### 语境一致性检查

| 风险类型 | 表现 | 示例 |
|---|---|---|
| 断章取义 | 只取有利部分，忽略限定条件 | 原文"在XX条件下可增长5%"，引用写成"增长5%" |
| 扩大范围 | 将局部结论当作全局结论 | 将"天然气管道"数据套用到"全部管道" |
| 偷换主体 | 将A企业的数据/结论归给B企业 | 将中石油数据归给国家管网 |
| 时间错位 | 用旧数据支撑新判断 | 用2020年数据论证2025年趋势 |
| 口径混淆 | 不同口径的数据直接比较 | 将"设计能力"与"实际输量"比较 |
| 因果倒置 | 将相关关系当作因果关系 | "X增长的同时Y增长"写成"X导致Y增长" |

### 语境验证方法

1. 定位引用在原文中的上下文（前后各2-3句）
2. 确认引用的限定条件、适用范围、前提假设
3. 检查报告中使用该引用的语境是否与原文一致
4. 不一致时：调整引用表述以反映原文限定条件，或降级为 `[待补充引用]`

---

## 五、链式验证

对于推导链条上的引用，逐环节验证：

```
原始数据 → 中间判断 → 结论
   ↓           ↓          ↓
验证存在    验证推导    验证结论
验证准确    验证逻辑    是否有据
```

如果中间环节存在未引用/无法验证的数据或假设，整个链条标记为"引用链不完整"，在台账中注明断点。

---

## 验证结论标记

| 结论 | 标识 | 含义 |
|---|---|---|
| 验证通过 | `[已验证]` | 四个层次全部通过，可以正式引用 |
| 验证通过（含提示） | `[已验证，注：...]` | 通过但有注意事项（如时效提示） |
| 无法独立验证 | `[待独立验证]` | 用户提供的内部数据或非公开报告 |
| 验证不通过 | `[引用存疑]` → 降级为 `[待补充引用]` | 四个层次任一不通过 |

---

## 批量验证优先级

当资源有限时，按以下优先级验证：

1. **P0**：报告核心论断所依赖的引用（数量少、影响大）
2. **P1**：精确数值引用（风险高、容易被挑错）
3. **P2**：政策性引用（文号、机关、文件名称）
4. **P3**：国际比较和行业趋势引用
5. **P4**：背景性和说明性引用
