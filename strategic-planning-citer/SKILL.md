# Strategic Planning Citer

为战略规划报告生成可溯源、可验证的引用体系。确保报告中每个事实性论断都有真实、可追溯的来源，杜绝虚假引用。

## Operating Stance

作为战略部的引用审核专员，对报告中的每一个数据、政策引用、市场判断进行溯源验证。核心原则：**宁可标注"待补充"，绝不编造引用**。

## When to Use

- 为已完成初稿的报告补充规范引用
- 审核现有报告引用是否真实可溯源
- 为报告生成引用汇总（文末参考文献列表）
- 在 `strategic-planning-writer` 完成后、`strategic-planning-reviewer` 之前运行
- 用户要求"检查引用""补充出处""确保数据有来源"时

## 管线位置

```
researcher → architect → writer → citer → reviewer
```

Citer 在 writer 输出初稿后介入，确保引用质量后再进入 reviewer 终审。

## 引用格式规范

### 正文内引用标注

正文中采用上标方括号编号 `【1】`，紧跟在被引用的事实、数据或论断之后。

格式：
```
截至2025年末，全国天然气管道总里程达到XX万公里【1】。根据国务院关于促进天然气协调稳定发展的政策要求，需加快储气库建设【2】。
```

编号从 【1】 开始，按在正文中首次出现的顺序依次递增。同一来源多次引用时，使用同一编号。

### 文末引用汇总

在报告末尾（reviewer 审查之前），生成「参考文献」汇总表。格式：

```
【1】国家统计局，《中国统计年鉴2025》，2025年。来源：知识库
【2】国务院，《关于促进天然气协调稳定发展的若干意见》（国发〔2018〕31号），2018年。来源：知识库
【3】国家能源局，《中国天然气发展报告2025》，2025年。检索自：https://www.nea.gov.cn/xxx.html
【4】国家管网集团，2025年年报。来源：用户提供
```

汇总表按编号顺序排列，每条包含：编号、来源全称（含文件名、文号）、发布年份、来源类型标注。

### 来源类型标注规则

| 来源类型 | 文末标注格式 | 说明 |
|---|---|---|
| 知识库已有资料 | `来源：知识库` | 数据从已上传的知识库中获取 |
| 官方网站检索获取 | `检索自：<完整URL>` | 标注检索到的具体网页链接，方便读者点击溯源 |
| 用户提供材料 | `来源：用户提供` | 内部数据或用户上传的文件 |
| 知识库 + 官网交叉验证 | `来源：知识库；交叉验证：<URL>` | 知识库数据经官网核实后标注 |

**重要**：对于通过 WebSearch 检索到的资料，文末必须附上标题超链接，格式为 `检索自：[标题](URL)`，确保读者可一键跳转到原始网页进行溯源。

## Workflow

### 1. 接收并扫描报告

接收报告草稿（全文或按章节），扫描以下内容：

- 所有数值型数据（里程、产能、投资额、增长率、年限等）
- 所有政策引用（"根据XX政策""XX文件指出""国家要求"等）
- 所有判断句（"预计""据分析""综合研判""中长期看"等）
- 所有比较句（"高于/低于行业平均水平""领先/落后于"等）
- 所有专有名词（项目名称、机构名称、指标体系、工程代号等）

### 2. 对每个事实性陈述进行分类

| 类型 | 标识 | 处理方式 |
|---|---|---|
| 已有规范引用 | `[已引用]` | 验证来源真实性、时效性 |
| 用户提供数据 | `[用户数据]` | 标注数据来源方，注明"待用户确认口径" |
| 可匹配知识库来源 | `[知识库可溯源]` | 从已上传知识库中匹配，文末标注"来源：知识库" |
| 可匹配公开来源 | `[可溯源]` | 通过 WebSearch 检索官方网站匹配，文末附标题超链接 |
| 常识性行业判断 | `[行业常识]` | 标注为无需特定引用的行业共识 |
| 无法验证的断言 | `[待补充引用]` | 先尝试 WebSearch 检索官方网站，找不到再标记 |

### 3. 先检索再留空

对于报告中标记为 `[待补充]` 或缺少引用的事实性数据：

1. **首先检索知识库**：检查已上传的知识库中是否有相关数据或来源
2. **其次 WebSearch 官方网站**：按以下优先级检索：
   - 政府网站（.gov.cn）：国务院、发改委、能源局、国家统计局
   - 央企官网：国家管网集团、中石油、中石化、中海油
   - 国际组织官网：IEA、EIA、World Bank 等
3. **检索到则补充引用**：将数据填入正文，并在文末附上检索来源的标题超链接
4. **检索不到再留空**：仅当知识库和官方网站均无法找到时，保留 `[待补充引用]`

### 4. 交叉验证引用真实性

对每一条引用执行验证规则：

- **存在性验证**：该文件/数据/讲话是否真实存在？
- **时效性验证**：引用是否在有效期内？（政策类一般不超过5年，市场数据一般不超过2年）
- **准确性验证**：引用的数字、表述是否与原文一致？
- **语境验证**：引用是否脱离原语境被曲解使用？

无法通过任一验证的引用，降级为 `[待补充引用]`。

### 5. 生成引用汇总

在报告末尾生成「参考文献」汇总，格式见上文「文末引用汇总」。

每条引用记录包含：
- 引用编号（【1】, 【2】, ...）
- 来源全称（含文件名、文号）
- 发布年份
- 来源类型标注（知识库 / 检索自URL / 用户提供）
- 对于 WebSearch 来源，附标题超链接

### 6. 输出

交付以下内容：
1. 标注引用后的报告正文（正文内【1】【2】标注 + 文末参考文献汇总）
2. 待补充引用清单（按优先级排列）
3. 引用质量评估（覆盖率、可靠率、风险点）

## 引用质量评估标准

| 指标 | 计算方式 | 合格线 |
|---|---|---|
| 引用覆盖率 | 已引用事实数 / 应引用事实总数 | ≥ 85% |
| 引用可验证率 | 已验证引用数 / 总引用数 | ≥ 90% |
| 权威来源占比 | 官方政府/央企来源数 / 总引用数 | ≥ 40% |
| 过期引用率 | 超过有效期的引用数 / 总引用数 | 0% |

## 核心规则

### 必须遵守

1. **绝不编造引用**。无法找到来源的断言，标记为 `[待补充引用：需提供...]`。
2. **每条引用必须有唯一编号**【1】【2】【3】...，按正文出现顺序编号。
3. **引用必须包含足够定位原文的信息**：来源名 + 年份 + 文件/报告名 + 可获取路径。
4. **区分来源类型**：知识库来源标注"来源：知识库"，WebSearch 来源标注"检索自：URL"并附标题超链接，用户提供材料标注"来源：用户提供"。
5. **检查时效性**。超过5年的政策引用需标注"注意时效性"；超过2年的市场数据需标注"需更新至最近年份"。
6. **检查语境一致性**。引用内容必须与原文语境一致，不得断章取义。
7. **数值引用必须带单位、口径、基准年**。不允许孤零零的数字。

### 禁止行为

- 禁止为 `[待补充]` 的数据编造任何来源
- 禁止将模糊来源（"业内人士""有关专家""相关数据显示"）作为引用
- 禁止引用不可追溯的网页快照、已失效链接、无法确认的内部文件
- 禁止为了覆盖率而降低引用标准
- 禁止引用非官方新闻网站和自媒体平台作为数据来源

### 数据检索规则

当知识库中缺失数据、需要从网络检索补充时：

1. **限定检索范围**：只能在 CLAUDE.md 中列出的 100 个白名单网站、.gov.cn 政府网站、央企官网、IEA/EIA/World Bank 等国际组织官网范围内检索。不在白名单内的网站，禁止检索和引用。
2. **交叉验证**：单一来源的数据标注"单一来源，未经交叉验证"。至少两个独立官方来源互相印证的数据方可标注为"已验证"。
3. **时效校验**：检索到的数据确认发布日期，超过2年的市场数据标注"数据偏旧，建议更新"。
4. **溯源到原始出处**：新闻报道中引用的数据，必须追溯到原始发布机构的官方文件或公告，不得以新闻报道作为最终引用来源。
5. **保留检索路径**：文末汇总中记录完整URL和检索日期，确保他人可复现检索过程。

## 与上下游 Skill 的接口

- **从 writer 接收**：带有 `[待补充：...]` 标记的报告草稿
- **从 researcher 接收（可选）**：evidence pack 中的事实表，用于引用匹配
- **向 reviewer 交付**：带有完整引用标注和参考文献汇总的报告，reviewer 只需审核内容质量

## 特殊情况处理

### 用户提供了内部数据

- 来源标注为"来源：用户提供，[数据类型]，[提供日期]"
- 验证状态标注为"用户确认后方可采用"

### 需要引用涉密或内部文件

- 不写出文件全文，仅标注文件类型和年份
- 例：`据集团内部[XX规划]（2024）`
- 汇总中标注"来源：内部文件，引用已脱敏"

### 引用链条不完整

当发现某一判断有引用，但其上游推导过程无引用时：
- 标记上游断言为 `[待补充引用]`
- 在汇总"备注"列注明引用链断裂位置
