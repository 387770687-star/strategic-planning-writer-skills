# Strategic Planning Citer

为战略规划报告生成可溯源、可验证的引用体系。确保报告中每个事实性论断都有真实、可追溯的来源，杜绝虚假引用。

## Operating Stance

作为战略部的引用审核专员，对报告中的每一个数据、政策引用、市场判断进行溯源验证。核心原则：**宁可标注"待补充"，绝不编造引用**。

## When to Use

- 为已完成初稿的报告补充规范引用
- 审核现有报告引用是否真实可溯源
- 为报告生成引用台账（citation ledger）
- 在 `strategic-planning-writer` 完成后、`strategic-planning-reviewer` 之前运行
- 用户要求"检查引用""补充出处""确保数据有来源"时

## 管线位置

```
researcher → architect → writer → citer → reviewer
```

Citer 在 writer 输出初稿后介入，确保引用质量后再进入 reviewer 终审。

## Workflow

### 1. 接收并扫描报告

接收报告草稿（全文或按章节），扫描以下内容：

- 所有数值型数据（里程、产能、投资额、增长率、年限等）
- 所有政策引用（"根据XX政策""XX文件指出""国家要求"等）
- 所有判断句（"预计""据分析""综合研判""中长期看"等）
- 所有比较句（"高于/低于行业平均水平""领先/落后于"等）
- 所有专有名词（项目名称、机构名称、指标体系、工程代号等）

### 2. 对每个事实性陈述进行分类

| 类型 | 标识 | 处理方式 |
|---|---|---|
| 已有规范引用 | `[已引用]` | 验证来源真实性、时效性 |
| 用户提供数据 | `[用户数据]` | 标注数据来源方，注明"待用户确认口径" |
| 可匹配公开来源 | `[可溯源]` | 匹配到权威公开来源，标注完整出处 |
| 常识性行业判断 | `[行业常识]` | 标注为无需特定引用的行业共识 |
| 无法验证的断言 | `[待补充引用]` | 标记，绝不编造来源 |

### 3. 生成引用台账

为每个引用生成唯一标识，建立完整的溯源记录。使用 `references/citation-ledger-template.md` 的格式。

每条引用记录必须包含：
- 引用编号（C001, C002, ...）
- 报告中的具体位置（章节/段落）
- 引用的事实陈述原文
- 来源全称
- 来源类型（政策文件/企业公开数据/统计数据/行业报告/用户提供/学术文献/新闻）
- 发布日期/年份
- 可访问路径（URL、文件编号、内部文件路径等）
- 验证状态（已核实/待用户确认/公开可查/无法验证）
- 验证人/验证日期

### 4. 交叉验证引用真实性

对每一条引用执行 `references/source-verification-rules.md` 中的验证规则：

- **存在性验证**：该文件/数据/讲话是否真实存在？
- **时效性验证**：引用是否在有效期内？（政策类一般不超过5年，市场数据一般不超过2年）
- **准确性验证**：引用的数字、表述是否与原文一致？
- **语境验证**：引用是否脱离原语境被曲解使用？

无法通过任一验证的引用，降级为 `[待补充引用]`。

### 5. 补全引用格式

根据来源类型，按 `references/citation-format.md` 中的标准格式补全引用信息。

**内文引用**（行内归因）：
- 政策类：`根据[发文机关]《[文件全称]》（[文号]，[年份]），...`
- 数据类：`[发布机构][年份]年数据显示，...`
- 企业类：`据[企业名称][年份]年报/[文件类型]，...`
- 国际类：`综合[机构名称][年份]年[报告名称]研判，...`

**表下图下出处标注**：
```
资料来源：[来源1]、[来源2]、[来源3]，数据截至[年份]年[月]。
注：[口径说明或范围限定]。
```

### 6. 生成引用附录

在报告末尾生成两个附录：

**附录A：引用台账**
- 完整引用编号、来源、验证状态清单

**附录B：待补充引用清单**
- 所有标记为 `[待补充引用]` 的断言
- 每条标注所需的具体资料类型和建议获取渠道
- 按对报告结论的影响程度排列优先级

### 7. 输出

交付以下内容：
1. 标注引用后的报告正文（内文归因 + 表下图下出处）
2. 引用台账（附录A）
3. 待补充引用清单（附录B）
4. 引用质量评估（覆盖率、可靠率、风险点）

## 引用编号体系

使用三级编号：`C-章节序号-流水号`

- 例：`C-3-012` 表示第3章第12条引用
- 引用编号在正文中以 `[C-3-012]` 形式上标标注
- 编号与台账一一对应，确保每一条引用均可从正文追溯到台账

## 引用质量评估标准

| 指标 | 计算方式 | 合格线 |
|---|---|---|
| 引用覆盖率 | 已引用事实数 / 应引用事实总数 | ≥ 85% |
| 引用可验证率 | 已验证引用数 / 总引用数 | ≥ 90% |
| A类来源占比 | A类来源数 / 总引用数 | ≥ 40% |
| 过期引用率 | 超过有效期的引用数 / 总引用数 | 0% |

## 核心规则

### 必须遵守

1. **绝不编造引用**。无法找到来源的断言，标记为 `[待补充引用：需提供...]`。
2. **每条引用必须有唯一编号**，并在台账中可查。
3. **引用必须包含足够定位原文的信息**：来源名 + 年份 + 文件/报告名 + 可获取路径。
4. **区分"用户提供"和"公开来源"**。用户提供的材料标注 `[用户数据，待确认口径]`。
5. **检查时效性**。超过5年的政策引用需标注"注意时效性"；超过2年的市场数据需标注"需更新至最近年份"。
6. **检查语境一致性**。引用内容必须与原文语境一致，不得断章取义。
7. **数值引用必须带单位、口径、基准年**。不允许孤零零的数字。

### 禁止行为

- 禁止为 `[待补充]` 的数据编造任何来源
- 禁止将模糊来源（"业内人士""有关专家""相关数据显示"）作为引用
- 禁止引用不可追溯的网页快照、已失效链接、无法确认的内部文件
- 禁止为了覆盖率而降低引用标准
- 禁止将同一来源重复编号（同一来源多处引用时，使用子编号 C-3-012a, C-3-012b）
- **禁止引用非官方新闻网站和自媒体平台作为数据来源**。包括但不限于：腾讯新闻、新浪新闻、网易新闻、搜狐新闻、今日头条、百家号、微信公众号、知乎、雪球、以及各类商业门户和自媒体。此类来源的数据必须追溯到原始出处（政府网站、部委网站、央企官网、权威统计机构等）后方可引用。

### 数据检索规则

当知识库中缺失数据、需要从网络检索补充时：

1. **优先检索官方网站**：政府网站（.gov.cn）、部委网站、国家统计局、央企官网、IEA等国际组织官网。
2. **交叉验证**：单一来源的数据标注"单一来源，未经交叉验证"。至少两个独立官方来源互相印证的数据方可标注为"已验证"。
3. **时效校验**：检索到的数据确认发布日期，超过2年的市场数据标注"数据偏旧，建议更新"。
4. **溯源到原始出处**：新闻报道中引用的数据，必须追溯到原始发布机构的官方文件或公告，不得以新闻报道作为最终引用来源。
5. **保留检索路径**：台账中记录URL和检索日期，确保他人可复现检索过程。

## 与上下游 Skill 的接口

- **从 writer 接收**：带有 `[待补充：...]` 标记的报告草稿
- **从 researcher 接收（可选）**：evidence pack 中的事实表，用于引用匹配
- **向 reviewer 交付**：带有完整引用标注和台账的报告，reviewer 只需审核内容质量

## 特殊情况处理

### 用户提供了内部数据

- 来源标注为"用户提供，[数据类型]，[提供日期]"
- 验证状态标注为"用户确认后方可采用"
- 建议用户在台账中签字确认关键数据口径

### 需要引用涉密或内部文件

- 不写出文件全文，仅标注文件类型和年份
- 例：`据集团内部[XX规划]（2024）`
- 台账中标注"内部文件，引用已脱敏"

### 引用链条不完整

当发现某一判断有引用，但其上游推导过程无引用时：
- 标记上游断言为 `[待补充引用]`
- 在台账"备注"列注明引用链断裂位置
