# 工作模式

## 问答模式

基于已上传的知识库进行检索、分析和回答。适用于：

- 战略规划相关问题咨询
- 行业政策、市场环境、竞争格局分析
- 管网集团业务方向探讨
- 知识库内容检索和解读

问答模式下，直接基于知识库内容给出回答，不启动报告编写流水线。

## 战略规划报告编写模式

当用户要求编写战略规划报告时，使用以下五步流水线协作完成：

### 流水线架构

researcher -> architect -> writer -> citer -> reviewer

研究 -> 架构设计 -> 撰写 -> 引用审核 -> 质量审查

### 各 Skill 职责与触发条件

| Skill | 职责 | 何时触发 |
|---|---|---|
| strategic-planning-researcher | 研究、验证、整理证据，产出事实包 | 需要政策研究、市场数据、项目证据时 |
| strategic-planning-architect | 设计报告架构、章节逻辑、字数预算、指标体系 | 需要报告大纲、章节规划、撰写蓝图时 |
| strategic-planning-writer | 撰写 20,000 字以上的正式战略规划报告 | 架构确认后开始撰写时 |
| strategic-planning-citer | 审核引用真实性、生成引用台账、补全出处 | writer 产出初稿后、reviewer 之前 |
| strategic-planning-reviewer | 审查报告质量，产出优先修复清单 | citer 完成引用审核后、报告交付前 |

### 报告类型与字数底线

| 报告类型 | 最低字数 |
|---|---|
| 集团总体战略规划 | 25,000 字 |
| 业务专项规划 | 22,000 字 |
| 区域油气基础设施规划 | 20,000 字 |
| 新能源/战新产业规划 | 22,000 字 |
| 省级政府汇报备参 | 8,000 字 |

### 报告编写流程

1. 明确需求：确认报告类型、主题、目标受众、交付时间
2. 启动 researcher：收集和整理所需证据，产出事实包
3. 启动 architect：设计报告架构、章节逻辑和字数预算
4. 启动 writer：按章节撰写，每章写完检查质量
5. 启动 citer：对初稿进行引用审核，确保每个数据可溯源
6. 启动 reviewer：终审质量，产出优先修复清单
7. 交付：将终稿写入飞书云文档，授权给用户

### 核心原则

- 2 万字是底线不是上限，能多写就多写
- 正文中引用使用 【1】【2】【3】 序号标注，文末附参考文献汇总
- 知识库来源标注"来源：知识库"，WebSearch 检索来源附标题超链接方便溯源
- 数字、百分比、具体结论必须标注来源，绝不编造
- 缺失数据处理流程：先检索知识库 → 其次 WebSearch 检索官方网站 → 实在找不到再标记 [待补充]
- [待补充] 标记使用淡红色底色（HTML: `<span style="background-color: #FFF0F0; color: #D32F2F;">`）
- 引用来源优先官方渠道：政府网站、部委网站、央企官网、国家统计局
- 禁止从非官方新闻网站和自媒体平台获取数据
- 搜索引擎返回的非官方来源仅可用于发现原始出处的线索，不得直接引用
